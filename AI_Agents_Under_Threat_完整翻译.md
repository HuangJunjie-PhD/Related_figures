## 摘要

**人工智能（AI）Agent** 是一种软件实体，能够基于预定义的目标和数据输入**自主执行任务**或**做出决策**。AI Agent 能够感知用户输入、推理和规划任务，并执行操作，在算法开发和任务性能方面取得了显著进展。然而，它们所面临的安全挑战仍然**探索不足**且**尚未解决**。本综述深入探讨了 AI Agent 所面临的新兴安全威胁，将其归类为**四个关键知识缺口**：

1. **多步用户输入的不可预测性**
2. **内部执行的复杂性**
3. **运行环境的可变性**
4. **与不受信任外部实体的交互**

通过系统性地审查这些威胁，本文重点强调了在保护 AI Agent 方面取得的进展和存在的局限性。本文提供的见解旨在激发进一步的研究，以解决与 AI Agent 相关的安全威胁，从而促进更**健壮**和**安全**的 AI Agent 应用的发展。

**CCS 概念**: • 安全与隐私 → 特定领域的安全和隐私架构

**关键词和短语**: AI Agent，可信度，安全性

**贡献说明**: Zehang Deng 和 Yongjian Guo 对本研究的贡献相等。

## 1 引言

AI Agent 是通过**自主性**、**反应性**、**主动性**和**社交能力**展现智能行为的计算实体。它们与环境和用户交互，通过**感知输入**、**推理任务**、**规划行动**以及使用**内部和外部工具**执行任务来实现特定目标。由大型语言模型（LLMs，如 GPT-4 [4]）驱动的 AI Agent 已经彻底改变了各种领域完成任务的方式，包括**医疗保健** [3]、**金融** [206]、**客户服务** [177] 和 **Agent 操作系统** [117]。这些系统利用 LLM 在推理、规划和行动方面的先进能力，使其能够以卓越的性能执行复杂任务。

尽管 AI Agent 取得了显著进展，但它们日益增加的复杂性也带来了新的安全挑战。确保 AI Agent 的安全性至关重要，因为它们被部署在各种关键应用中。**AI Agent 安全性**指的是旨在保护 AI Agent 免受可能损害其功能、完整性和安全的漏洞和威胁的措施和实践。这包括确保 Agent 能够安全地处理用户输入、执行任务以及与其他实体交互，而不易受到恶意攻击或意外有害行为的影响。这些安全挑战源于**四个知识缺口**，如果不加以解决，可能导致漏洞 [32, 104, 121, 211] 和潜在滥用 [140]。

如图 1 所示，AI Agent 中的四个主要知识缺口是：

**图 1. AI Agent 安全中知识缺口的说明。这些知识缺口增加了 AI Agent 的安全挑战。具体而言，缺口 1 与感知威胁（第 3.1 节）相关，缺口 2 与大脑威胁（第 3.2 节）和行动威胁（第 3.3 节）相关，缺口 3 与 Agent2Environment 威胁（第 4.1 节）相关，缺口 4 涉及 Agent2Agent 威胁（第 4.2 节）和记忆威胁（第 4.3 节）。**

以下各点将详细阐述这些知识缺口：

---

### 缺口 1：多步用户输入的不可预测性

用户在与 AI Agent 交互中起着关键作用，不仅在任务启动阶段提供指导，还通过他们的多轮反馈影响整个任务执行过程中的方向和结果。用户输入的多样性反映了不同的背景和经验，指导 AI Agent 完成众多任务。然而，这些多步输入也带来了挑战，尤其是当用户输入描述不充分时，可能导致潜在的安全威胁。用户输入的规范不足不仅会影响任务结果，还可能引发一系列意外反应，导致更严重的后果。此外，存在恶意用户故意引导 AI Agent 执行不安全的代码或操作，这增加了额外的威胁。因此，确保用户输入的清晰性和安全性对于 AI Agent 的有效和安全运行至关重要。

---

### 缺口 2：内部执行的复杂性

AI Agent 的内部执行状态是一个复杂的链-环结构，从提示的重格式化到 LLM 规划任务以及使用工具。许多内部执行状态是隐式的，难以观察详细的内部状态。这导致许多安全问题无法及时检测到的威胁。AI Agent 安全性需要审计单个 AI Agent 的复杂内部执行。

---

### 缺口 3：运行环境的可变性

在实践中，许多 Agent 的开发、部署和执行阶段跨越各种环境。这些环境的可变性可能导致行为结果的不一致。例如，负责执行代码的 Agent 可能会在远程服务器上运行给定代码，可能导致危险操作。因此，在多个环境中安全地完成工作任务是一个重大挑战。

---

### 缺口 4：与不受信任外部实体的交互

AI Agent 的一个关键能力是教大模型如何使用工具和其他 Agent。然而，当前 AI Agent 与外部实体之间的交互过程假设受信任的外部实体，导致广泛的实际攻击面，如**间接提示注入攻击** [56]。AI Agent 与其他不受信任实体交互具有挑战性。

尽管已经做出了一些研究工作来解决这些缺口，但专注于 AI Agent 安全性的全面综述和系统分析仍然缺乏。一旦弥合这些缺口，AI Agent 将受益于改善的任务结果（由于更清晰、更安全的用户输入）、针对潜在攻击的增强安全性和鲁棒性、各种运行环境中的一致行为，以及用户增加的信任和可靠性。这些改进将促进 AI Agent 在关键应用中的更广泛采用和集成，确保它们能够安全有效地执行任务。

现有的关于 AI Agent 的综述 [93, 114, 172, 203, 229] 主要关注其架构和应用，而没有深入探讨安全挑战和解决方案。本综述旨在通过提供 AI Agent 安全性的详细审查和分析来填补这一空白，识别缓解这些威胁的潜在解决方案和策略。本文提供的见解旨在激发进一步研究，以解决与 AI Agent 相关的安全威胁，从而促进更健壮和安全的 AI Agent 应用的开发。

在本综述中，我们基于四个知识缺口系统性地审查和分析 AI Agent 安全的威胁和解决方案，涵盖广度和深度方面。我们主要收集了顶级 AI 会议、顶级网络安全会议和高引用 arXiv 论文的论文，时间跨度从 2022 年 1 月到 2024 年 4 月。包括的 AI 会议（但不限于）：NeurIPs、ICML、ICLR、ACL、EMNLP、CVPR、ICCV 和 IJCAI。包括的网络安全会议（但不限于）：IEEE S&P、USENIX Security、NDSS、ACM CCS。

本文组织如下。第 2 节介绍 AI Agent 的概览。第 3 节描述与缺口 1 和缺口 2 相关的单 Agent 安全问题。第 4 节分析与缺口 3 和缺口 4 相关的多 Agent 安全性。第 5 节提供该领域发展的未来方向。

## 2 AI Agent 概览

在本节中，我们在**统一概念框架**上呈现 AI Agent 框架的概览及其安全挑战和知识缺口。

### 2.1 统一概念框架上的 AI Agent 概览

#### AI Agent 定义

在本综述中，我们将 AI Agent 的定义精炼为其本质特征：在动态环境中**自主和迭代**地感知反馈和行动的能力。虽然独立的 LLM 可以处理文本输入并生成输出，但将此类模型视为"Agent"会导致过于宽泛的解释，与现有的 LLM 安全综述 [215, 232] 重叠。相反，我们强调**集成 LLM 作为其"大脑"**的 AI Agent，能够与环境迭代交互、处理反馈并实时决策。

我们对 AI Agent 的定义广义上包括：

1. **基于 LLM 的 Agent** [203]：涵盖多模态能力（例如，处理视觉和其他模态，通常称为 MLLM Agent [204]）
2. **基于 RL 的 Agent** [48, 143]：利用强化学习通过环境交互优化策略并最大化累积奖励

这两种范式相互补充，形成一个全面的框架，其中 LLM 贡献多模态推理和规划，而 RL 增强自适应决策和持续改进。这种协同作用突出了 AI Agent 的更广泛概念，其中关键能力包括实现目标、获取知识（通过 LLM）和持续改进（通过 RL）。为了解决这些方面带来的独特风险，我们在第 4.1.2 节中分析与 RL 相关的风险，在第 3.2 节中分别分析与 LLM 相关的风险。此外，我们在第 3 和第 4 节中详述的四个缺口中检查高级多模态数据引入的风险。

#### Agent 结构

AI Agent 的结构可以分为两个核心组件：**内部执行**和**交互**，如图 2 所示。

**图 2. AI Agent 的一般工作流程。通常，AI Agent 由三个组件组成：感知、大脑和行动。**

1. **内部执行（Intra-execution）**：指单 Agent 架构的内部功能，包括三个关键过程：
   - **感知（Perception）**：利用输入格式化器通过**提示工程**增强用户多模态输入，为大脑提供精炼的输入
   - **大脑（Brain）**：集成大型语言模型的推理和规划能力，分析信息、推导逻辑结论并制定决策策略
   - **行动（Action）**：在大脑的推理和规划指导下，通过调用外部工具来执行子任务

   迭代过程（①、②）简化了内部执行。

2. **交互（Interaction）**：涉及 Agent 与外部实体的参与，例如与其他 Agent 协作、检索记忆或利用外部工具和环境数据。值得注意的是，由于检索过程中的相关安全风险，记忆被视为外部资源。感知、大脑和行动的联合过程形成了内部执行的基础，而交互将 Agent 的能力扩展到更广泛的系统。

Li 等人 [93] 未能同时考虑多 Agent 场景的能力和安全性。一项研究 [172] 仅提供了科学 LLM Agent 固有的潜在风险。Zhang 等人 [229] 仅调查了 AI Agent 的记忆机制。
